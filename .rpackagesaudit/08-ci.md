## Objective
Ensure the standard CI workflows exist and reference the correct branch: R-CMD-check and pkgdown deploy.

## Preconditions
- 04-cran-check is done.
- `<branch>` known from STATE.md.

## Steps
1. **R-CMD-check.** If `.github/workflows/R-CMD-check.yaml` is missing, create it from the `r-lib/actions` template with the standard matrix (macOS release, Windows release, ubuntu devel/release/oldrel-1; `r-lib/actions/check-r-package@v2`). If present, only verify the trigger branches include the real default `<branch>` (`main` vs `master`).
2. **pkgdown deploy.** If `.github/workflows/pkgdown.yaml` is missing, create it from the `r-lib/actions` template that builds the site and deploys to the `gh-pages` branch on push to `<branch>`.
3. **Branch correctness.** Across all workflow files, replace any hardcoded `main`/`master` that does not match the detected `<branch>`:
   ```bash
   grep -rn 'branches:' .github/workflows/ || true
   ```
4. Validate YAML parses:
   ```bash
   for f in .github/workflows/*.y*ml; do \
     Rscript -e 'q <- commandArgs(TRUE); yaml::read_yaml(q[1]); cat(q[1], "OK\n")' "$f" || echo "$f INVALID"; done
   ```
   (Install `yaml` first if absent.)
5. Add `^\.github$` to `.Rbuildignore` if not already present.

## Validation
- `.github/workflows/R-CMD-check.yaml` and `pkgdown.yaml` exist and parse as valid YAML.
- Trigger branches match the detected `<branch>`.

## Outputs
- CI workflow files under `.github/workflows/`.

## On failure
- If `yaml` cannot be installed, fall back to a structural `grep` for required keys (`on:`, `jobs:`, `steps:`) and report that full validation was skipped.
